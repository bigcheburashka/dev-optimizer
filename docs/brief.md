# Agent Brief

## Что строим

**Dev Optimizer** — CLI-инструмент для анализа JS/TS-репозиториев, который помогает быстро находить и исправлять:
- waste в CI/CD,
- проблемы в dependencies,
- неоптимальности в Docker.

## Что НЕ строим в MVP

Не включать в MVP:
- глубокий bundle intelligence,
- full security product,
- сложную license policy систему,
- историю/perf dashboard,
- enterprise-платформу,
- GitLab parity любой ценой.

## Почему этот фокус правильный

Широкий список идей уже перекрыт зрелыми тулзами. Монетизируется не "ещё один анализатор", а:
- единый workflow,
- приоритизация,
- ROI,
- safe autofix,
- интеграция в PR/CI.

## Что важно для успеха

1. Мало фич, но высокое качество.
2. Минимум ложных срабатываний.
3. Понятный вывод без "воды".
4. Demo за 5 минут на реальном репо.
5. Возможность быстро сделать первую value delivery.

## Целевой результат MVP

На типовом JS/TS-репозитории инструмент должен:
- находить 5–10 понятных улучшений;
- показывать ожидаемый эффект;
- выдавать единый человекочитаемый report;
- безопасно применять только low-risk fixes.
